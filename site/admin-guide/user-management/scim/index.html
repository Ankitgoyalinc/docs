Enable SCIM UMA Authentication

This is step by step guide to configure UMA for oxTrust and SCIM client. High level architecture overview is available in article OX SCIM Architecture Overview

Base configuration: create oxAuth clients, policies, ...

  1. Register oxAuth client with scope “http://docs.kantarainitiative.org/uma/scopes/prot.json”. Property “oxAuthTokenEndpointAuthMethod” of this client should has value “client_secret_basic”. It's possible to do that using few methods: Client Registration, using oxTrust GUI, manually add entry to LDAP. oxTrust will use this oxAuth client to obtain PAT. Sample result entry:

oxAuthTrustedClient: true

  2. Register oxAuth client with scope “http://docs.kantarainitiative.org/uma/scopes/authz.json”. Property “oxAuthTokenEndpointAuthMethod” of this client should has value “client_secret_basic”. It's possible to do that using few methods: Client Registration, using oxTrust GUI, manually add entry to LDAP. SCIM Client will use this oxAuth client to obtain AAT. Sample result entry:

oxAuthTrustedClient: true

  3. Add UMA scope. These are list of steps which allows to add new scope.

  4. Log with administrative privileges into oxTrust.
  5. Open menu “OAuth2→UMA”.
  6. Select “Scopes” tab and click “Add Scope Description”.
  7. Select “Internal” type.
  8. Fill the form.

  9. Click “Add” button. Sample result entry:

        dn: inum=@!1111!D386.9FB1,ou=scopes,ou=uma,o=@!1111,o=gluu
    objectClass: oxAuthUmaScopeDescription
    objectClass: top
    displayName: Access SCIM
    inum: @!1111!D386.9FB1
    owner: inum=@!1111!0000!D9D9,ou=people,o=@!1111,o=gluu
    oxId: access_scim
    oxRevision: 1
    oxType: internal

  10. Create UMA policy. These are list of steps which allows to add new policy.

  11. Log with administrative privileges into oxTrust.
  12. Open menu “OAuth2→UMA”.
  13. Select “Policies” tab and click “Add Policy”.
  14. Select language “Python”.

  15. Paste this base policy script:

        from org.xdi.oxauth.service.uma.authorization import IPolicyExternalAuthorization
    class PythonExternalAuthorization(IPolicyExternalAuthorization):
    def authorize(self, authorizationContext):
    print "SCIM policy. Attempting to authorize SCIM client"
    client_id = authorizationContext.getGrant().getClientId()
    print "SCIM policy. SCIM client:", client_id
    print "SCIM policy. Client is authorized"
    return True

  18. Register UMA resource set. It's possible to do that via Rest API or via oxTrust GUI. Sample code: https://svn.gluu.info/repository/openxdi/oxAuth/trunk/Client/src/test/java/org/xdi/oxauth/ws/rs/uma/RegisterResourceSetFlowHttpTest.java. These are list of steps which allows to add new resource set.

  19. Log with administrative privileges into oxTrust.
  20. Open menu “OAuth2→UMA”.
  21. Select “Resources” tab and click “Add Resource Set”.
  22. Fill the form.
  23. Add UMA Scope which we created in previous steps.
  24. Add Client which we created in second step.

  25. Click “Add” button. Sample result entry:

oxTrust configuration (Resource Server)

oxTrust UMA related configuration properties from oxTrust.properties:

uma.issuer=http://localhost:8080/oxauth/seam/resource/restv1/oxauth/uma-configuration
uma.client_id=@!1111!0008!F781.80AF
uma.client_password=<encrypted_password>
uma.resource_id=1403179695657
uma.scope=http://localhost:8080/oxTrust/ws/scope/access_scim

Values of these properties correspond to entries from first section.

SCIM Client (Requesting Party) sample code

This is sample SCIM Client code which request user information from server.

package gluu.scim.client.dev;

import gluu.scim.client.ScimClient;
import gluu.scim.client.ScimResponse;

import javax.ws.rs.core.MediaType;
 public class TestScimClient {
public static void main(String[] args) {
ScimClient scimClient = ScimClient.umaInstance("http://localhost:8080/oxTrust/seam/resource/restv1", "http://localhost:8080/oxauth/seam/resource/restv1/oxauth/uma-configuration", "@!1111!0008!FDC0.0FF5", "client_password");
try {
ScimResponse response1 = scimClient.retrievePerson("@!1111!0000!D9D9", MediaType.APPLICATION_JSON);
        System.out.println(response1.getResponseBodyString());
} catch (Exception ex) {
ex.printStackTrace();
}
}

}

Values from these example correspond to entries from first section.
